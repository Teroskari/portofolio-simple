Buses of Turku

android, node, open data

This was a solo project I started working on late autumn of 2015. An android application that visualizes the locations of local bus-traffic on a google-maps element.

The application receives its data from a NodeJS server that in turn follows the open data provided by the city of Turku.

Keep-A-Head

android, node, thingsee

A project built during a December 2015 hackathon focused on IoT solutions. Our team crafted a smart-helmet companion application using the data of a Thingsee One. The application provides weather and speed data as well as logs possible crashes. The idea was to automatically call an ambulance after a crash should certain conditions be met.

We built a Node rest-api using Hapi to provide data to the application. The Thingsee One was linked to deliver data to our server using Thingsee's Web Api.

PL 2015

meteor, phonegap, html, css, js

This project was a second screen application for a large student event. The app was built with MeteorJS and was released for both android and iOS platforms. The platform was still in beta and we encountered quite a few problems. In hindsight, native applications would have been easier.

While partaking in development, I also co-ordinated the project with the people from the Pikkulaskiainen event's organizers and handled interaction with the client.

Kroniikkamaatti

react, node, firebase, heroku

I've been actively involved in my student organization since I began my studies. The organization paper desired a web app allow for first year students to write nice things about each other for a sping 2016 release. I volunteered to built such an application.

I settled on ReactJS to get more comfortable with the framework. I settled on Firebase as a simple back-end that works for this specific project. The project was then hosted on heroku for access.
